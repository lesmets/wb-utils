How it works

Enter your team or company name, pick a brand color, and optionally add a logo. Then select the tools your team actually uses and the languages you want to ship. The preview shows exactly what a packaged page will look like. When you click create, the packager fetches a neutral, stripped-down template of each selected tool, applies your branding, and assembles everything into a single ZIP, all locally in your browser.

The ZIP contains one folder per tool, each holding a self-contained index.html plus its translations and favicons, and a landing page at the top that links the tools you included. There is nothing to install and nothing to configure: unzip it onto any static web server, intranet host, or file share and it works. Because every page runs entirely in the browser and makes no network requests, the package is a good fit for environments where data must not leave the building.

The packaged pages are the same tools that run on wbest.app, minus everything that only makes sense online: no analytics, no search-engine markup, no article text, just the tool. Each page keeps a one-line attribution to WBest Tools in the footer.

Frequently asked questions

What does the package contain?

A folder per tool with a single self-contained index.html (plus its translations and favicons), a landing page linking the tools you picked, and your logo if you added one. The pages are the same tools that run on wbest.app, stripped of everything except the tool itself and rebranded with your name and color.

How do I host the package?

Unzip it anywhere a browser can reach: any static web server, an intranet host, a company file share, or even a USB stick. There is no build step, no database, and no configuration. Opening index.html directly from disk works too.

Is my logo or company name uploaded anywhere?

No. The packager runs entirely in your browser: it fetches the neutral tool templates, applies your branding locally, and assembles the ZIP on your device. Your name, color, and logo never leave your browser.

Do the packaged tools need an internet connection?

No. Every packaged page is fully self-contained, makes no network requests, and keeps working offline. That is what makes them safe for sensitive data on an internal network.

Am I allowed to self-host these tools with my own branding?

Yes, that is exactly what this packager is for. Each generated page keeps a small "Thank you for using WBest Tools" line in the footer linking back to wbest.app. Please leave that attribution in place.
